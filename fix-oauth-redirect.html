Fix OAuth Redirect URLs

Problem: OAuth Redirect URL Mismatch

Error: "both auth code and code verifier should be non-empty"

This error means the OAuth redirect URL is not configured correctly in either Supabase or Google Cloud Console.

Required Redirect URLs

Supabase Site URL:
http://localhost:3000
Supabase Redirect URL:
https://xylaoshplmsevlxzizrd.supabase.co/auth/v1/callback
Google Cloud Console Redirect URI:
https://xylaoshplmsevlxzizrd.supabase.co/auth/v1/callback

Step 1: Fix Supabase Settings

  1. Go to Supabase Authentication Settings
  2. Click on "Authentication" > "Providers" > "Google"
  3. Update these settings:
    • Site URL: http://localhost:3000 (NOT https!)
    • Redirect URLs: https://xylaoshplmsevlxzizrd.supabase.co/auth/v1/callback
    • ❌ IMPORTANT: Do NOT add http://localhost:3000/auth/callback to redirect list

Step 2: Fix Google Cloud Console

  1. Go to Google Cloud Console Credentials
  2. Find your OAuth 2.0 Client ID for Southside RP
  3. Click "Edit" or the pencil icon
  4. Under "Authorized redirect URIs", add:
    • https://xylaoshplmsevlxzizrd.supabase.co/auth/v1/callback
  5. Remove any incorrect redirect URIs (like localhost:3000/auth/callback)
  6. Click "Save"
